## 7. 実施手順（案A: r5.2xlarge → r5.xlarge）

> ⚠️ インスタンスクラス変更は再起動を伴い、**数分間の接続断** が発生する。  
> メンテナンスウィンドウ（JST 土曜 00:30〜01:00）での実施を推奨。  
> Readerから先に変更し、動作確認後にWriterを変更する。

```bash
REGION="ap-northeast-1"

# Step 1: instance-2（Reader）を先に変更（リスク低）
aws --no-cli-pager rds modify-db-instance \
  --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-2 \
  --db-instance-class db.r5.xlarge \
  --apply-immediately

# Step 2: instance-2 が available になるまで待機
aws rds wait db-instance-available \
  --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-2
echo "instance-2 変更完了 → アプリ動作確認"

# Step 3: 問題なければ instance-1（Writer）を変更
aws --no-cli-pager rds modify-db-instance \
  --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-1 \
  --db-instance-class db.r5.xlarge \
  --apply-immediately

aws rds wait db-instance-available \
  --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-1
echo "instance-1 変更完了"

# Step 4: 変更後の確認
aws --no-cli-pager rds describe-db-instances \
  --region $REGION \
  --filters "Name=db-cluster-id,Values=ksm-posprd-db-cluster" \
  --query 'DBInstances[*].{ID:DBInstanceIdentifier,Class:DBInstanceClass,Status:DBInstanceStatus}' \
  --output table
```

**変更前の確認事項:**
- [ ] アプリケーション側の DB 接続先（Writer / Reader エンドポイント利用状況）を確認
- [ ] カスミ担当者・USMH側に変更スケジュールを通知
- [ ] メンテナンス時間帯にバッチ処理が重ならないことを確認（バッチは JST 21:25、01:25）
- [ ] スナップショットを手動取得してから実施する
