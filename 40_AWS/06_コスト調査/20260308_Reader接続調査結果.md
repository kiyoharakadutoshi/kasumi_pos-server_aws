## 5. コストダウン判断

### ✅ 確実に実施可能: instance-2 ダウンサイズ

| 対応 | 内容 | 月削減 | リスク |
|---|---|---|---|
| **instance-2: r5.2xlarge → r5.large** | 接続数は最大1、読み取り専用Lambda4本のみ | **~$507/月** | **低**（監視系Lambdaは軽量処理） |
| **instance-1: r5.2xlarge → r5.xlarge** | Writeエンドポイント、常時10〜19接続 | **~$507/月** | **低**（CPU最大0.015%） |

**合計削減: 月 $1,014 / 年 $12,168**（オンデマンドのまま）

### ⚠️ 要検討: instance-2 の削除

Readerを削除すると、Readerエンドポイント（`cluster-ro-...`）へのリクエストは**自動的にWriterにフォールバック**するAurora仕様のため、アプリ動作上は問題ない可能性が高い。

ただし以下のリスクがある：
- `prd/Replica_Kasumi_RO` を使うLambdaがWriterに接続されるため、監視系処理の分離ができなくなる
- フェイルオーバー時の自動昇格先がなくなる

**推奨: 削除ではなくダウンサイズ（r5.large）に留める。**

## 6. 推奨対応プラン（最終版）

```
【Step 1】両インスタンスを r5.2xlarge → r5.large にダウンサイズ
  月削減: $1,014/月  年間削減: $12,168

【Step 2】1〜2ヶ月後に問題なければ Reserved Instance 1年購入
  さらに月削減: ~$355/月
  ────────────────────────────────
  最終合計: 月$1,369削減、年間$16,428削減（約62%削減）
  現状 $2,208/月 → $839/月
```

## 7. 変更コマンド（r5.2xlarge → r5.large）

```bash
REGION="ap-northeast-1"

# instance-2（Reader）を先に変更
aws --no-cli-pager rds modify-db-instance \
  --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-2 \
  --db-instance-class db.r5.large \
  --apply-immediately

aws rds wait db-instance-available --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-2
echo "instance-2 完了 → 動作確認"

# instance-1（Writer）を変更（数分の接続断あり）
aws --no-cli-pager rds modify-db-instance \
  --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-1 \
  --db-instance-class db.r5.large \
  --apply-immediately

aws rds wait db-instance-available --region $REGION \
  --db-instance-identifier ksm-posprd-db-instance-1
echo "instance-1 完了"
```
