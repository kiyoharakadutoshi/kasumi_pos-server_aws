# AWS 月次コスト妥当性チェック手順

## 実行タイミング
**毎月1〜3日**（Cost Explorerの反映に1〜2日の遅延があるため、月末ではなく翌月初旬に実施）

| 実施月 | 対象データ | 推奨実施日 |
|--------|-----------|-----------|
| 4月初旬 | 3月分コスト | 4/1〜4/3 |
| 5月初旬 | 4月分コスト | 5/1〜5/3 |
| 6月初旬 | 5月分コスト | 6/1〜6/3 |

## 実行手順

### 1. CloudShell を開く
[AWS CloudShell](https://ap-northeast-1.console.aws.amazon.com/cloudshell/home?region=ap-northeast-1) を開く

### 2. スクリプトを貼り付けて実行
```bash
# リポジトリからスクリプトを取得して実行
curl -s https://raw.githubusercontent.com/kiyoharakadutoshi/kasumi_pos-server/main/40_AWS/06_%E3%82%B3%E3%82%B9%E3%83%88%E8%AA%BF%E6%9F%BB/monthly_cost_check.sh | bash

# または直接スクリプトをコピー&ペーストして実行
bash monthly_cost_check.sh
```

### 3. 結果の保存
実行結果を以下のファイル名でこのディレクトリに保存：
```
YYYYMM_コスト確認結果.md
例: 202603_コスト確認結果.md
```

## チェック項目と判断基準

| # | チェック項目 | 正常範囲 | 要注意 |
|---|------------|---------|--------|
| 1 | 月間合計コスト | $2,000〜$2,300 | $2,500超 |
| 2 | RDS（r5.2xlarge×2） | ~$2,027/月 | 急増時は調査 |
| 3 | RDS CPU平均 | <5% | 50%超 |
| 4 | RDS 最大接続数 | <100 | 500超 |
| 5 | Lambda エラー率 | 0% | 1%超 |
| 6 | 前月比 | ±10%以内 | +10%超 |

### ダウンサイズ承認後（r5.large移行後）の正常範囲
| 項目 | 正常範囲 |
|------|---------|
| 月間合計コスト | $700〜$900 |
| RDS（r5.large×2） | ~$658/月 |

## 異常時の対応フロー

```
月次コストが閾値超過
    │
    ├─ +10〜20%超 → サービス別内訳を確認（【2】の出力）
    │               新規リソースの追加有無を確認
    │
    ├─ RDS CPU高騰 → スロークエリログ確認
    │                CloudWatch Insights でクエリ分析
    │
    ├─ Lambda エラー増加 → CloudWatch Logs で原因調査
    │                      StepFunctions 実行履歴確認
    │
    └─ S3 急増 → ライフサイクルポリシー確認
                 不要ファイルの棚卸し
```
