## [1] RDS（両環境で全体の60%占有）— ❌ 妥当ではない

### コスト実績

| 環境 | 月額 | 構成 | 実CPU利用率 |

| PRD | $2,218/月 | r5.2xlarge×2 + t3.medium×2 | Writer 0.015%、Reader 1.69% |
| STG | $2,261/月 | 同等構成 | 同程度と推定 |
| **合計** | **$4,479/月** | | |

### 問題点

- CPU使用率が最大0.015%（Writerの8vCPU中、ほぼゼロ）
- Write IOPSが通常3〜5、バッチ時でも最大62。db.r5.largeで十分対応可能
- ストレージは5.65GBしかない（64GB RAMの9%未満）
- **必要スペックの4〜8倍の過剰スペック**

### 削減見込み（aws_006で計画済み）

| 施策 | PRD月削減 | STG月削減 | 年間削減 |
|---|---|---|---|
| r5.2xlarge→r5.xlarge（案A） | $1,004 | $1,004 | **$24,096** |
| +RI 1年購入（案C） | $1,425 | — | **$17,103（PRDのみ）** |

### 判定

調査完了・妥当。改修指示書aws_006の承認を急ぐべき。

## [2] Transfer Family（両環境で15%占有）— ⚠️ やや非妥当

### コスト実績

| 環境 | 月額 | 構成 |
|---|---|---|
| PRD | $554/月 | 3台（OC/SG/SH）24時間稼働 |
| STG | $554/月 | 同上 |
| **合計** | **$1,108/月** | |

### 問題点

- SH系は2025/11に追加→これがコスト急増原因（$432→$669/月）
- SH系の利用は2件/日のみ（60件/月・平均38MB/件）
- 転送量課金は月$0.07〜$0.16（ほぼ無視できる）
- 3台を1台に統合すれば月$432×2環境=$864削減（年$10,368）が可能

| 案A: SH系停止 | $432（両環境） | $5,184 |
| 案B: 3台→1台統合（推奨） | $864（両環境） | **$10,368** |

### 判定

調査完了・妥当。SH系の業務要否をカスミ様に確認中。

## [6] STG > PRD コスト逆転現象 — ❌ 要対応

**STG($3,776) > PRD($3,614)で月$162高い。本来STGはPRDより安いべき。**

### 原因内訳

| 原因 | 月額差 |
|---|---|
| STGのEC2が4台（PRDは2台） | +$96/月 |
| STGのALBが2台（PRDは0台） | +$14/月 |
| STGのS3が急増（2月: STG $72 vs PRD $5） | +$67/月（2月） |
| STGのVPC（ALB用ENI追加分） | +$44/月 |
| PRDのセキュリティサービス（STGは無効） | -$74/月 |

### S3急増の原因

- 2月にSTGのS3コストが$72に急増（9月は$1.5）
- 42_P003_historyテーブル肥大化（1億行32GB）→ バックアップサイズ増大が原因の可能性
- aws_021で42_P003_history縮小後にS3コストも正常化する見込み

### 判定

EC2台数差は構成上やむを得ないが、S3急増はP003_history肥大化と連動。aws_021実施で解消見込み。
