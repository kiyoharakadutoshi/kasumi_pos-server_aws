## VangleTek指摘内容と対応

### 指摘① AWS CLIコマンドの手順不足
**指摘:** コード差分セクション（AWS CLIコマンド）に以下のステップが欠けている
- KMS Key 作成
- IAM Role for AWS Backup 作成
- Aurora Cluster を Backup Plan に紐付け（Backup Selection）

**対応:**
- ステップ1〜6を全て追加
- IAM Role名はプロジェクト命名規則に準拠: `ksm-{env}-iam-role-backup`（AWSDefaultServiceRole は使用しない）
- KMS alias も命名規則に準拠: `alias/ksm-{env}-kms-backup`

### 指摘② 改修範囲とテスト範囲の不整合
**指摘:** 改修一覧はPRDのみ、テスト手順はTEST→STG→PRDで不整合

**背景（Luvinaの自己分析）:**
- 初版作成時点でDEV・STG環境の調査が未完了だったことが原因

**対応:**
- 対象環境を **DEV（スパイク）/ STG / PRD** 全3環境に拡張
- 実施順序を以下に統一:
  - ① DEV（スパイク）環境で確認 → ② STG環境で確認 → ③ 顧客承認 → ④ PRD本番反映
- テスト手順表に環境別バナー行を追加

**環境情報（ログ確認済）:**

| 環境 | AWSアカウント | prefix | RDSクラスター |
|---|---|---|---|
| DEV（スパイク） | 891376952870 | ksm-posspk- | posspk-db |
| STG | 750735758916 | ksm-posstg- | ksm-posstg-db-cluster |
| PRD | 332802448674 | ksm-posprd- | ksm-posprd-db-cluster |

### 指摘③ コスト見積もり（$20〜30/月）の根拠なし
**指摘:** VangleTekの試算は ~$5.47/月（現状 ~6GB 使用時）

**議論の経緯:**
- 「将来ストレージが増えれば$20〜30になる」という説明を検討
- ストレージ増加要因として「棚札対応店舗の大幅増加」が考えられるが、根拠のある将来予測ではない
- AWS Backupは差分バックアップのため、ストレージ増加 ≠ バックアップコスト増加
- 仮に100倍（~600GB）になった場合: ~$448/月 となり、$20〜30とは全く異なる
- **結論: `$20〜30/月` は根拠なし → 完全削除し `~$5.47/月` のみ記載**
