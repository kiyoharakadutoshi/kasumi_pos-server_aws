【前回の判定】問題なし — 開発環境限定（localhost）であるため。
【Đánh giá trước】Không vấn đề — chỉ dành cho development (localhost).

【訂正理由】ソースコード調査で以下が判明。
【Lý do đính chính】Điều tra source code phát hiện:

(1) 文書記載の `/gift`, `/gift-api` → `:8080` はソースコードに存在しない（文書誤記）
(1) Mô tả `/gift`, `/gift-api` → `:8080` trong tài liệu không tồn tại trong source code (lỗi tài liệu)

(2) **API URLがビルド時にハードコード**されている:
(2) **API URL được hardcode lúc build**:
- `npm run build-stg` → `https://api-stg.ignicapos.com/api/v1`（STG）
- `npm run build` / `build-spk` → `https://api-spk.ignicapos.com/api/v1`（PRD）
- `--env input` の指定を誤ると、**STGフロントエンドがPRDのバックエンドAPIを呼び出す**。
- Nếu chỉ định sai `--env input`, **frontend STG sẽ gọi API backend PRD**.
→ **環境独立性（②）に違反。**
→ **Vi phạm độc lập môi trường (②).**

(3) production版（いなげや用）は `http://172.172.1.105:8080` をプライベートIP直指定。環境変数化されていない。
(3) Bản production (cho Inageya) trỏ trực tiếp IP private `http://172.172.1.105:8080`. Chưa biến môi trường hóa.
→ **環境独立性（②）・情報漏洩防止（③）に違反。**
→ **Vi phạm độc lập môi trường (②) và ngăn chặn rò rỉ (③).**

(4) web-fe EC2の内部構成（nginx設定等）は未確認。要VPN接続で調査継続。
(4) Cấu hình bên trong web-fe EC2 (nginx, v.v.) chưa xác nhận. Cần kết nối VPN để tiếp tục điều tra.

【対応案】
【Đề xuất sửa】
- API URLをビルド時ハードコードからランタイム環境変数（window.__ENV__等）に変更
- Đổi API URL từ hardcode lúc build sang biến môi trường runtime (window.__ENV__, v.v.)
- production版のIP直指定をDNS名に変更
- Đổi IP trực tiếp của bản production sang tên DNS
